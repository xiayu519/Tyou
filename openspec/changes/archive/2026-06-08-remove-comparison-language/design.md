## Context

## Goals / Non-Goals

**Goals:**

- README 的项目定位只描述 Tyou 是 Cocos Creator 3.8.7 + TypeScript 客户端框架。
- 工作流文档只保留“非源码材料需复核”这个通用约束，不保留具体来源标签。
- memory 不记录无必要的材料位置。
- 已归档 change 里的相关表述也同步清理，避免后续搜索误导。

**Non-Goals:**

- 不评估或修改开发者已有 `ty-framework` 代码改动。
- 不新增新的工作流概念。

## Decisions

- **文档只写本项目事实**：README 和规则入口不写来源对照，只写当前实现、节点契约和开发流程。
- **复核规则保留但去来源化**：保留“非源码材料不能直接写成项目事实”的原则，避免以后再次把参考材料原文搬进项目文档。
- **删除上一轮 reference memory**：该信息可从当前对话和归档恢复，不值得长期进入项目记忆。

## Risks / Trade-offs

- **Risk: 清理过度导致工作流少了上下文** → Mitigation: 保留“非源码材料复核”这一条足够通用的规则。
- **Risk: archive 记录被改动** → Mitigation: 只清理措辞，不改变 tasks、验证结果和归档事实。
