# 小游戏 Shader 性能边界

## 首要风险

- 透明 overdraw：UI、Spine、序列帧图通常都是透明渲染，同屏大面积叠加比单次 ALU 更危险。
- 纹理采样：描边、帧间混合、噪声叠加都会增加采样数。
- 多 pass：小游戏端优先避免，除非收益明确且有真机验证。
- 动态分支和循环：低端移动 GPU 上不稳定，循环次数必须固定且很小。
- 材质实例数量：大量实例会增加内存和状态切换成本。

## 预算建议

| 效果 | 建议预算 |
| --- | --- |
| 灰度/染色/闪白 | 1 次主贴图采样 |
| 流光 | 1 次主贴图采样，尽量不用额外贴图 |
| 溶解 | 1 次主贴图 + 1 次噪声/遮罩采样 |
| 简单描边 | 主贴图 + 4 方向 alpha 采样；8 方向需谨慎 |
| 序列帧无混合 | 1 次主贴图采样 |
| 序列帧帧间混合 | 2 次主贴图采样，只给重点特效使用 |
| UV 扰动 | 主贴图 + 1 次噪声/扰动图采样 |

## 精度

- 颜色、UV、小范围参数优先低精度思路，但以 Cocos Effect 实际语法和平台编译为准。
- 阈值、溶解边缘、SDF 距离、循环终止条件用更稳的精度。
- 不为了省一点 ALU 牺牲明显视觉稳定性。

## 降级策略

- 描边：8 方向 -> 4 方向 -> 关闭。
- 溶解：程序噪声 -> 噪声贴图 -> 简单 alpha 裁切。
- 序列帧：帧间混合 -> 无混合。
- 扭曲：动态噪声 -> 静态扰动贴图 -> 关闭。
- 大面积 UI：材质特效 -> 组件颜色/动画。

## 输出必须说明

- 目标同屏数量预估。
- 每像素采样数。
- 是否会增加透明层或全屏覆盖。
- 是否需要每帧更新材质参数。
- 低配关闭条件。
