# Spine Shader

## 适用效果

- 角色/怪物受击闪白。
- 中毒、冰冻、灼烧等整体染色。
- 选中描边、阵营描边、交互高亮。
- 死亡溶解、召唤出现、护盾遮罩。
- 简单 UV 扰动或噪声边缘。

## Spine 特有注意

- Spine 通常使用 atlas 贴图和 slot 顶点色，shader 必须保留顶点色/alpha。
- 先确认 premultiplied alpha 设置；闪白和描边如果忽略预乘 alpha，容易出现黑边或白边。
- Spine 网格会随骨骼变形，适合做屏幕/UV/颜色效果，不适合假设固定矩形边界。
- 多 slot/多材质的 Spine 对象，运行时改参数要确认改到所有相关材质实例。

## 推荐实现层级

1. **颜色类**：闪白、染色、灰度，最低风险，优先做。
2. **阈值类**：溶解、遮罩，需要噪声/遮罩贴图和进度参数。
3. **描边类**：需要 alpha 邻域采样或额外描边资源，采样预算最高。
4. **扭曲类**：小幅 UV 偏移可做，强扭曲要小心 atlas 邻帧/邻图串色。

## 闪白公式思路

- 采样主贴图得到 `baseColor`。
- 保留 alpha。
- `rgb = mix(baseColor.rgb, flashColor.rgb, flashAmount)`。
- 输出再结合顶点色和透明度。

## 描边策略

- 小尺寸 Spine 可尝试 4 方向 alpha 采样。
- 中大型角色优先美术提供描边资源、膨胀贴图或后处理方案；不要默认 8/16 次采样。
- 若角色在战斗中数量多，描边应支持关闭或低配降级。

## 运行时绑定

- 受击/状态效果由战斗表现层设置材质参数，结算逻辑不要直接依赖 Cocos 材质对象。
- 高频状态变化避免每帧创建材质实例；初始化时缓存实例和参数名。
- 释放 Spine 节点时遵守资源引用计数和当前 UI/战斗对象释放流程。
