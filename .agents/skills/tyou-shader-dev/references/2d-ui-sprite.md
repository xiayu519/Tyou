# 2D UI / Sprite Shader

## 适用效果

- 灰度、禁用态、变暗、变亮。
- 受击闪白、染色、双色调。
- 外描边、内描边、边缘高亮。
- 流光、扫光、选中边框。
- 溶解、裁切、遮罩过渡。
- UV 扰动、水波、热浪、噪声闪烁。

## 默认策略

1. 优先单 pass。
2. 优先 1 张主贴图 + 0/1 张噪声或遮罩贴图。
3. 保留顶点色和 alpha：输出颜色通常要乘 `vertex color` 与贴图 alpha。
4. UI 大面积透明会产生 overdraw，效果强度优先用参数控制，不要堆多个透明层。
5. 如果只是纯颜色/透明/灰度，优先用组件颜色或材质参数，不急着写复杂 shader。

## 常用参数

- `_Color`：整体颜色或 tint。
- `_Intensity`：效果强度，范围 0-1。
- `_Progress`：溶解/裁切/流光进度。
- `_EdgeWidth`：描边/溶解边宽。
- `_EdgeColor`：边缘颜色。
- `_NoiseTex`：溶解/扰动噪声图。
- `_TimeScale`：动画速度，由脚本驱动时更可控。

## 描边建议

- 小图标描边可用 4/8 方向采样 alpha，采样数要明确。
- 大面积 UI 或列表 item 不建议每像素 8+ 次采样。
- Spine 或序列帧描边先确认 alpha 边界质量，必要时美术提供 SDF/描边贴图。

## 流光建议

- 使用 UV 与 `_Progress` 计算斜向条带即可，通常不需要额外贴图。
- 条带宽度、柔边、强度都做成参数。
- 列表里大量节点同时流光时，考虑错开相位或减少更新频率。

## 溶解建议

- 优先噪声贴图 + `_Progress`，不要在 shader 内做多层 fbm。
- 边缘色只在阈值附近混合。
- 透明裁切会改变边缘 aliasing，小游戏端要真机看锯齿。

## 运行时绑定

- UI 脚本中改材质参数前确认材质是否实例化，避免改到共享资源。
- 多个节点同效果不同进度时，每个节点需要独立材质实例或独立 uniform 通道。
- 动态贴图仍按 `tyou.res` 和 UI 自动释放规则管理。
