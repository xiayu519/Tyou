## AI 工作流入口过长导致关键约束被忽略

- 现象：双 CLI 工作流中，非 Codex 入口内容过长后，任务分级、OpenSpec 门禁等关键规则容易被模型忽略。
- 根因：同一套项目规则在多个 CLI 适配壳中维护，入口上下文变长且需要双向同步，增加了执行漂移和维护成本。

- 二次修正：通过 OpenSpec change `streamline-codex-workflow` 继续删除过渡期解释和排除式表述，让入口只写 Codex 如何执行。
- 后续规则：单工作流的目标是精确执行；入口文档优先写正向步骤和硬约束，不写历史兼容背景。
