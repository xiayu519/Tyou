# 2026-05-13 问题记录

## openspec PowerShell wrapper 被执行策略拦截

- 现象：在 PowerShell 中执行 `openspec list --json` 报 `无法加载文件 ...\openspec.ps1，因为在此系统上禁止运行脚本`。
- 定位：npm 安装的 PowerShell shim 受当前系统 ExecutionPolicy 限制。
- 根因：PowerShell 默认策略禁止加载 `openspec.ps1`，但同目录的 `openspec.cmd` 可正常运行。
- 修正动作：本仓库后续在 PowerShell 环境优先使用 `cmd /c openspec.cmd ...` 执行 OpenSpec 命令。

## PSD `.img` 后缀必须优先于智能对象展开

- 现象：`.img` 智能对象或图层组如果先进入智能对象/组合递归解析，会被拆成内部节点，违背“整图导出”的预期。
- 定位：`Psd2CCC-Digest.jsx` 原本只对 `LayerSet` 的 `.img` 做合图，`ArtLayer` 智能对象会先走 `tryExpandSmartObject`。
- 根因：`.img` 是视觉合成强规则，优先级应高于文本解析、智能对象展开和普通图层分支。
- 修正动作：在 `ArtLayer` 分支入口优先处理 `.img` 并导出 PNG；`.img` 导出失败只记录 warning 并跳过，不再回落为普通拆分。

## Photoshop CC 2018 未知数据弹窗不能只靠 DialogModes.NO

- 现象：展开智能对象时仍反复弹出“此文档包含未知数据，将抛弃未知数据以保持图层可编辑...”对话框。
- 定位：弹窗来自 Photoshop 打开智能对象内部文档的模态提示，`placedLayerEditContents` 的 `DialogModes.NO` 和全局 `app.displayDialogs = DialogModes.NO` 在 CC 2018 上不一定生效。
- 根因：这是 Photoshop 文件兼容性/未知数据提示，不完全受脚本 action dialog mode 控制。
- 修正动作：曾尝试导出期间启动 Windows-only watcher 自动点击“保留图层”，但实机会造成电脑卡顿，已从 `Psd2CCC-Digest.jsx` 回滚。后续遇到该弹窗手动点击“保留图层”，不要再引入轮询/自动点击 watcher。
