# 2026-05-23

## PSD2CCC 直方图去重误合并不同 PNG

- 现象：`师祖赐福(1)-AutoSlice.psd` 导出后，`img_masterlove_once` 在结构 JSON 中复用了 `img_masterlove_five` 的 PNG，四个卡片九宫格也出现类似错复用，Cocos 生成图看起来像多出/错叠了一张图。
- 定位：`Psd2CCC-Digest.jsx` 的 PNG 去重注释写“文件内容哈希”，实际用 `tmp.histogram` 加尺寸生成指纹；直接用 `psd_tools` 读取 PSD 原层时，`img_masterlove_once` 与 `img_masterlove_five` 像素 SHA256 不同。
- 根因：直方图只描述颜色数量分布，不描述像素位置或最终 PNG 内容；同尺寸、同字体/同色系 UI 图很容易误判重复。
- 修正动作：change `fix-psd-dedup-fingerprint` 中把普通图层与 `.img` 组合导出去重改为保存临时 PNG 后读取二进制内容计算指纹，并在命中哈希时做全量二进制比对。
- 二次修正：实测 Photoshop 重新导出后 `dedup: 0`，说明整 PNG 文件字节指纹过严，可能被 PNG 元数据或编码细节打穿；改为只保留影响图像的 PNG chunk 计算 `pngVisualHash`，排除时间/文本等非图像元数据。
