## PSD2CCC 智能对象展开压扁/偏移

- 现象：`qtb.psd` 中未打 `.img` 的智能对象展开后，在 Cocos 中出现图标压扁、碎片偏移、表现不贴近 PSD；隐藏智能对象内文字后仍存在。
- 定位：`Client/tools/psd/Psd2CCC-Digest.jsx` 的 `makeSmartObjectTransform()` 原先用智能对象内容文档画布 `smartDoc.width/height` 作为内部映射基准。
- 根因：部分智能对象内部画布包含透明边距，外层智能对象图层 bounds 对应的是内部真实可见像素范围，不是整张内容文档画布；用画布尺寸映射会把展开子层错误缩放/偏移。剪贴蒙版和混合语义仍可能造成拆层后视觉差异，但不是这次压扁的唯一原因。
- 修正动作：change `fix-psd-smart-object-transform` 中改为优先复制智能对象文档并合并可见图层，计算内部可见 bounds；失败时退回可见图层 bounds union，再失败才用文档画布并记录 warning。

## PSD2CCC 智能对象包络变形不能拆节点还原

- 现象：`qtb.psd` 重新导出后火图标尺寸接近，但视觉表现仍不一致；中间宝箱大小和方向明显不对。
- 定位：用 `psd_tools` 直接解析 `qtb.psd`，`宝箱/宝箱` 的 placed layer transform 是 `(286,362)-(434,491)`，并带 `customEnvelopeWarp`；当前 `qtb-structure.json` 没有 transform/warp 字段，只能生成轴对齐 Cocos 节点。
- 根因：Photoshop 智能对象的 warp/包络变形、旋转、透视不能通过当前拆层节点树表达；强行展开会丢失外层变换。火图标另有大量剪贴蒙版 warning，拆成独立 PNG 后同样会丢失 Photoshop 合成关系。
- 修正动作：`Psd2CCC-Digest.jsx` 增加智能对象 placed-layer 检测，发现 `warp/customEnvelopeWarp` 或非轴对齐 transform 时不展开，回退为单张 PNG，并在 report 记录 `smartObjectTransform` warning；打开智能对象后若发现剪贴蒙版/调整层，也回退 PNG 并记录 `smartObjectComposite` warning。
