# 编辑器分析工具必须显式启停

## Feedback

- 会扫描、索引或持续监听项目资源的编辑器工具不得在扩展 `load()` 时默认激活。
- 工具应由开发者明确开启后才扫描并维护优化缓存；明确关闭后必须停止监听、释放内存并取消对项目的持续影响。
- 打开/关闭工具窗口和开启/关闭后台索引功能应是两个独立生命周期。

## Why

- 默认扫描会在开发者没有使用工具时占用 AssetDB 和编辑器资源，也会把工具查询失败误表现为项目启动错误。
- 参考工具采用显式开启、缓存复用、开启期间随资源刷新、关闭后无影响的模型。

## How To Apply

- 新增 Creator 资源分析、依赖扫描、重复资源检查或类似常驻工具时，默认状态为 disabled。
- `load()` 最多读取小型状态/缓存，不发起全量扫描；资源广播在 disabled 状态直接返回。
- 提供明确的“开启功能”和“关闭功能”，关闭时清理计时器、订阅、内存索引与用于表示开启状态的缓存。
- 只有功能处于 enabled 状态时，资源增删改广播才能调度刷新。

## Recheck

- 如果某个工具必须随项目启动自动运行，需要先说明不可延迟的原因、性能预算和关闭/回滚方式，并由开发者确认。
